# copilot-cli

## エージェントの使い分け

| シナリオ | 使用するエージェント |
|---|---|
| 小さい課題（1タスク相当）をすぐ解決したい | `quick-orchestrator` |
| 複数タスクにわたる課題を計画・管理しながら解決したい | `backlog-manager` でバックログ作成 → `orchestrator` で実行 |

### quick-orchestrator を使うとき

- タスクをバックログに分解するほどでもない小さい課題
- 例: 「このファイルのXXXを修正して」「設定ファイルにYYYを追加して」
- `orchestrator` と同じ進行管理の型をベースに、単一タスクだけを対象として委譲・レビュー・報告を行う

### orchestrator を使うとき

- 複数のタスクに分解される課題
- 複数日かかる規模の実装
- 依存関係のある複数タスクを管理しながら進めたい場合
- まず `backlog-manager` でバックログを作成し、次に `orchestrator` に対象バックログファイルを指定して実行する
