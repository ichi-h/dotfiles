# Notify スキル

## 概要

タスク完了時やユーザーへの確認が必要な場合に、`NOTIFY_WEBHOOK_URL` 環境変数に設定された Webhook URL へ通知を送信する。

## 通知タイミング

以下のタイミングで通知を送信する:

- タスクが完了したとき
- ユーザーへの確認・判断を促す必要があるとき

## 通知フォーマット

### タイトル

**固定**: `ワイの仕事終わったで！`

### 本文

- **必ず関西弁で記述すること**（最重要要件）
- 非常に簡潔に「何が完了したか」または「何を確認してほしいか」を伝える

## 通知コマンド

```bash
NOTIFY_BODY="<本文>"
[ -n "$NOTIFY_WEBHOOK_URL" ] && \
  jq -n --arg body "$NOTIFY_BODY" \
     '{"content": ("# ワイの仕事終わったで！\n\n" + $body)}' | \
  curl -H "Content-Type: application/json" \
       -X POST \
       --data-binary @- \
       "$NOTIFY_WEBHOOK_URL"
```

- `<本文>` を直接シェル文字列に埋め込むことは禁止する。必ず変数に代入してから `jq` 経由で渡すこと。

## 本文の例

| 状況 | 本文の例 |
|---|---|
| リファクタリング完了 | `リファクタのタスク終わったんで確認してや！` |
| PR作成完了 | `PRつくっといたで、レビューよろしゅう！` |
| テスト失敗・確認依頼 | `テストこけてもうたんで見てくれへん？` |
| 設計レビュー依頼 | `設計案まとめたんで、ええか確認してや！` |
| ビルドエラー・判断依頼 | `ビルドエラー出たで、どないしたらええか教えてや！` |

## 重要な注意事項

1. **本文は必ず関西弁**で、丁寧すぎず自然な話し言葉にする
2. 本文は一言〜二言程度に収める（冗長にしない）
