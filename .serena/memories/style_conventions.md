# Style and Conventions

## Code Style
- Nix: Standard Nix formatting
- Markdown: Used for agent definitions and skill files

## Agent Definition Format
- YAML frontmatter with `name`, `description`, `tools`, `model`
- Markdown body with structured instructions
- Standard Japanese for all artifact text

## Naming
- kebab-case for file names
- Agent files: `{name}.agent.md`
- Skill files: `SKILL.md` in skill directories
